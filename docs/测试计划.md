# 测试计划

## 后端

- 注册、登录、当前用户、退出。
- 注册必须先获取邮箱验证码；未带验证码、错误验证码、过期验证码和重复使用验证码均应失败。
- 忘记密码通过邮箱验证码重置；旧密码不能登录，新密码可以登录。
- SMTP 未配置时应用仍可启动，发送验证码接口返回明确业务错误。
- 患者可以预测并生成历史记录。
- 医生可以查看历史记录。
- 管理员可以查看模型指标和审计日志，所有 HTTP 操作审计记录应尽量包含 `clientIp`；带 `X-Forwarded-For` 时取第一个有效 IP，无代理头时回退请求远端地址。
- 数据库迁移执行后旧审计日志应被清空，新操作重新写入审计记录。
- 管理员上传数据集，校验缺少 `label` 的数据集会标记为未通过。
- 管理员创建训练任务后可以查询状态、训练历史和失败原因。
- 管理员可以保存评估记录并完成模型反馈的新增、编辑、删除。
- 管理员上传 `txt/pdf/docx` 知识文档后可以查看、下载并看到图谱构建状态；非管理员访问 `/api/admin/documents` 返回 403。
- 图谱接口覆盖健康检查、任务列表、搜索和可视化响应；MockMvc 默认不依赖真实 Neo4j，真实 Neo4j 集成测试可在提供 `MEDRISK_TEST_NEO4J_URI` 后单独执行。
- 图谱构建单元测试需要覆盖增量构建包含 `未构建` 与 `构建失败` 文档、全量重建中单文档失败不影响其他文档、单篇非空文档抽取结果不少于 50 个文档相关节点，以及 Neo4j 不可达时健康检查返回 `DOWN`。
- 疾病信息和病历 CRUD 覆盖单图、多图保存、患者/医生只读检索和管理员权限校验。
- 会话创建、消息发送、问答历史和患者只能访问自己会话的权限需要覆盖；缺少 LLM API Key 时兜底答案必须可用并保留教学演示/不能替代医生诊断的免责声明，同时返回 `usedModel`、`provider`、`fallbackUsed`、`chatMode`、`retrievalStatus`、`evidenceSources`。
- 智能问答需要分别验证日常聊天和医学问答：`chatMode=daily` 不执行检索，`chatMode=medical` 返回检索状态和证据来源。医学问答中的明显无关问题必须返回 `usedModel=policy-guard`、`provider=medrisk-policy`、`fallbackUsed=true`，并确认不会调用外部模型。
- 演示数据包导入需要验证幂等性：重复调用 `/api/admin/data-seeds/import` 后疾病、文档、病历和数据集不按重复次数膨胀，状态接口至少达到约 50 个疾病、100 篇文档、50 个合成病历和 5 组数据集。
- 权限分离需要覆盖患者无法访问医生草稿、管理员数据导入、图谱管理和数据集管理；医生无法发布、删除或触发全量重建；管理员可访问全部管理能力。
- 可见性过滤需要覆盖 `PUBLIC`、`DOCTOR_ONLY`、`ADMIN_ONLY`、`DRAFT` 在文档、疾病、病历、问答检索和图谱可视化中的后端过滤。
- 非管理员访问 `/api/admin/**` 返回 403。
- 报告生成接口返回报告 ID，下载接口返回 PDF。

## 模型服务

- 五类病种均返回风险等级、概率、置信度、Top 5 因素、建议、免责声明。
- 不支持的病种返回 404。
- 极端输入不会导致服务崩溃。
- 使用小型 CSV fixture 完成真实 XGBoost 训练，验证 `model.joblib`、`history.json`、指标和训练曲线。
- `GET /models/capabilities` 需要覆盖 XGBoost 可用、TabPFN/TabICL 依赖缺失时不可用原因，以及 mock 高级模型 adapter 的训练预测链路。
- 评估接口返回 Accuracy、Precision、Recall、F1、AUC、混淆矩阵和样本预测摘要。
- 激活训练模型后，对应病种预测返回新的模型版本号。

## 前端

- 页面全部中文显示。
- 登录页和顶部导航展示湖北工业大学与琢朴品牌资源。
- 提交预测后能看到风险卡、概率、图表、建议和报告入口。
- 风险大屏应使用 ECharts 平面动态图表渲染风险分布、病种分布、趋势和模型指标；图谱可视化仍使用 Three.js 三维力导向布局。
- 历史、报告、管理、智能问答、文档管理、图谱管理、图谱可视化、疾病信息、病历管理页面存在空状态和错误状态；导航不再出现“AI 辅助分析”。
- 患者、医生、管理员三种角色导航分别只显示允许模块：患者不显示文档管理、疾病信息、病历管理、图谱/数据集/用户管理，医生显示只读图谱可视化和草稿提交入口，管理员显示数据源管理、图谱管理和全部管理后台；侧边栏不再出现“个人中心”，顶部头像进入“个人信息”。
- 左侧栏固定在视口内，主内容独立滚动；管理员导航按分类折叠/展开，侧栏可收起并持久化状态。
- 浏览器标签页标题需要随当前用户和当前页面变化，格式为 `MedRisk AI - 角色 - 姓名/用户名 - 页面`。
- 管理员登录后 Neo4j 健康状态应持续轮询，离开图谱管理页再进入用户管理等页面时不应丢失已连接状态。
- 文档、疾病、病历页面需要展示公开/医生专用/草稿、来源标识；图谱可视化需要支持来源、可见性筛选和节点文字标签开关。
- 数据源管理页需要渲染导入统计、来源列表、最近导入记录和失败空状态。
- 管理后台不再显示横向 Tabs；模型版本、数据集管理、训练任务、模型评估、模型反馈、审计日志由左侧分类导航进入并渲染对应内容。
- 训练任务表单应显示“模型类型”，XGBoost 可选；TabPFN/TabICL 未安装高级依赖时禁用并展示原因。
- 审计日志表和管理员最近审计列表应显示“登录 IP”列，非登录记录显示 `-`。
- 数据集上传表单、训练任务表、训练曲线、评估指标和反馈表单均为中文界面并保留免责声明。

## 集成

1. 启动本地 Neo4j，确认 `bolt://localhost:7687` 可连接；若未启动，`.\scripts\start-all.ps1` 应在启动后端前给出明确失败信息，并说明不会自动连接 `med_tencent`、不应开放远端 `7687`。
1.1 若本地 `.env` 或当前环境误写 `MEDRISK_NEO4J_URI=bolt://neo4j:7687`，`.\scripts\check-env.ps1` 应提示该值只属于 Docker Compose；`.\scripts\start-all.ps1` 应改用 `bolt://localhost:7687` 并继续执行本机 Neo4j 可达性检查。
2. 本地演示执行 `.\scripts\start-all.ps1`，确认脚本返回后终端仍可继续输入命令，并且输出 `MedRisk AI services are ready`。
3. 确认前端按 `start-all.ps1` 输出地址可访问（通常 `http://localhost:5173`，本机 5173 被占用或被系统保留时为 `http://localhost:5241`）、后端 `http://localhost:8080/api/health`、模型服务 `http://localhost:8090/health`、Neo4j Browser `http://localhost:7474` 均可访问；脚本不得在健康检查未通过时宣称启动完成。
4. 确认启动脚本终端输出不包含 `Requirement already satisfied` 等 pip 依赖明细；模型服务依赖安装详情应写入 `logs/model-service-install.log`。
5. 确认模型服务进程使用 conda `MedRisk` 环境中的 Python，例如 `D:\IDEA\Anaconda\envs\MedRisk\python.exe` 或其他电脑上解析到的 `...\envs\MedRisk\python.exe`，而不是 `medrisk_model_service\.venv` 或随机 `PATH` Python。
6. 执行 `.\scripts\check-env.ps1`，确认输出 `Demo Flyway guard: OK`，并能看到当前 `Neo4j URI`、本地 Bolt 端口状态和 Docker daemon 可用性说明；若本机存在旧的 `data/medrisk-demo` H2 文件库，确认 `demo` profile 可跳过 Flyway checksum 历史差异并正常启动，该验证不适用于 Docker Compose/MySQL 部署。
7. 检查 `logs/backend.out.log`，Flyway 对本地 H2 应显示 schema up to date 或 no migration necessary，不应出现 checksum validation failed。
8. 直接请求 `POST http://localhost:8080/api/auth/login`，使用 `admin / 123456`，确认返回 `code: 0` 和 JWT；这是防止“后端未启动导致前端登录 500”复发的必测项。
8.1 配置 SMTP 后请求 `/api/auth/register/code` 和 `/api/auth/password/forgot`，确认真实邮箱可以收到验证码；未配置 SMTP 时确认返回明确错误且 `/api/health` 仍为 `UP`。
9. 再次执行 `.\scripts\start-all.ps1`，确认不会重复拉起模型服务、后端或前端进程；当 5173 不可绑定且前端运行在 5241 等 fallback 端口时，脚本必须复用已有端口，不得继续启动 5484、5500、5600、3000 等额外 Vite 实例。
10. 执行 `.\scripts\stop-all.ps1`，确认 5173、8080、8090 端口释放，且本地外置 Neo4j 不被停止。
11. 重新执行 `.\scripts\start-all.ps1`。
12. 登录 `admin / 123456`。
13. 在“数据源管理”执行导入或重建演示包，确认疾病、文档、病历、数据集数量达到中等演示规模，并且来源列表包含 MedlinePlus、Synthea、Disease Ontology/HPO、UCI、NHANES 等公开来源元数据。
14. 打开图谱管理和图谱可视化，确认 Neo4j 写入后首屏有足够节点/关系，并可按来源和可见性筛选。
15. 上传知识文档，执行单文档同步或增量构建，确认图谱任务完成且图谱可视化出现节点/关系；单篇非空文档应至少产生 50 个文档相关节点。
16. 人为保留一个 `构建失败` 文档后执行增量构建，确认该文档会被重新处理；批量任务中某篇失败时其余文档仍继续构建。
17. 新增疾病信息和病历，确认患者只能看公开摘要，医生可看公开和医生专用内容，管理员可编辑发布。
18. 使用智能问答发起文字问题、图片输入和输出图片请求，确认纯文本默认走 `MEDRISK_LLM_MODEL`，图片输入自动走 `MEDRISK_VISION_MODEL`，输出图片只在开关打开时调用 `MEDRISK_IMAGE_MODEL`；医学问答展示相关实体、疾病引用、病历引用，并区分公开可见和医生专用引用。
19. 完成一次心脏病预测。
20. 查看预测历史。
21. 生成并下载 PDF 报告。
22. 上传含 `label` 列的 CSV 数据集。
23. 创建训练任务并查看训练曲线。
24. 启用训练成功的模型版本。
25. 使用数据集评估模型并新增一条模型反馈。
26. 再次发起对应病种预测，确认使用训练模型版本。

## CI 验证

1. 执行 `.\scripts\ci.ps1`。
2. CI 应先停止本项目本地服务，避免 Vite/esbuild 锁定 `node_modules`。
3. 后端 `mvn test` 应通过。
4. 后端默认使用 H2 与不可达 Neo4j 测试配置，验证图谱接口降级响应和问答兜底，不要求真实 Neo4j；真实 Neo4j 集成验证单独配置 `MEDRISK_TEST_NEO4J_URI`。
5. 模型服务应使用 conda `MedRisk` 环境执行 `python -s -m pytest`。
6. 前端依赖缺失时应自动补装，随后 `npm run test` 和 `npm run build` 应通过。
7. 任一原生命令失败时，CI 应返回非零退出码。
8. 执行 `docker compose config`，确认 Linux Compose 配置有效；可用 Docker 时验证前端容器访问 `/api/health` 能经 Nginx 代理到后端。
9. 扫描 `README.md`、`docs/`、`scripts/` 和 `medrisk_backend/src/main/resources`，确认没有冲突标记、旧 `Resolve-CondaPython`、本地 `.venv` 回退说明，且 `validate-on-migrate: false` 只出现在 `application-demo.yml` 和解释性文档中。

## 可移植性验证

1. 在新电脑安装 Java 17、Maven、Node.js 20+ 和 Anaconda/Miniconda。
2. 创建 conda 环境 `MedRisk`；如果 conda 根目录不在常见位置，设置 `$env:MEDRISK_CONDA_ROOT`。
3. 执行 `.\scripts\check-env.ps1`，确认 `MedRisk model Python` 指向 `...\envs\MedRisk\python.exe`。
4. 执行 `.\scripts\start-all.ps1`，确认三项健康检查 200，且终端没有被后台服务占用。

## 本轮 UI 与模型回归

- 风险大屏顶部不得再出现“首页 / 在线预测 / 数据表格”胶囊导航。
- 登录页和用户界面 Logo 不应再使用白底 tile；透明 Logo 应自然融入背景，且不重复堆叠。
- 登录输入区域桌面端应至少占据约三分之一视口宽度，输入框和按钮高度应明显大于旧版。
- 登录页演示账号按钮应存在唯一选中态，选中/未选中的背景、边框、文字和 `aria-pressed` 明显不同；切换登录/注册/重置后清空选中态。
- ZP logo 应使用蓝色透明版本，登录页和侧栏不应出现白底或黑底 logo 块。
- `GET /models/public-evaluations` 应返回 CDC BRFSS 2024、CDC NHANES 2017-March 2020、UCI Heart Disease 与 UCI Chronic Kidney Disease 的公开评估来源、样本量和链接。
- `scripts/prepare-large-datasets.ps1` 应生成 `data/processed/manifest.json`、训练 CSV、评估 CSV 和来源元数据；管理员“导入公开训练数据集”应按 `sourceRecordId` 幂等注册公开数据集。
- 模型版本表应显示“评估数据集”列；默认基线应显示公开数据源，管理员启用训练模型后应显示已部署训练模型。
- 训练任务表单切换 `xgboost`、`tabpfn`、`tabicl` 时应显示不同超参数控件；旧请求不带 `hyperparameters` 时仍按 XGBoost 默认参数运行。
- 选择同病种 `evaluationDatasetId` 创建训练任务后，任务和模型版本应返回 `hyperparameters`、`evaluationDatasetName`、`evaluationDatasetSource`、`evaluationDatasetUrl`。
- 启用训练模型后应生成 `MEDRISK_ACTIVE_MODELS_FILE`，模型服务重启后对应病种预测仍使用该模型版本。
