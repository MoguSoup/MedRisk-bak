# 部署说明

## Docker Compose

```powershell
copy .env.example .env
docker compose up --build
```

生产环境必须修改 `.env` 中的数据库密码、Neo4j 密码和 JWT 密钥。Compose 部署会启动 MySQL、Neo4j、模型服务、后端和前端；后端默认连接 `bolt://neo4j:7687`。

Compose 会将本地 `uploads/` 和 `models/` 同时挂载给 Spring Boot 后端与 FastAPI 模型服务：

- 数据集：`uploads/datasets/`
- 知识文档、问答图片、疾病图片、病历图片：`uploads/` 下对应 bucket
- 训练产物：`models/training/`
- 报告：Docker 命名卷 `medrisk_reports`
- Neo4j 图谱数据：Docker 命名卷 `medrisk_neo4j_data`

## 本地开发端口

| 服务 | 端口 |
|---|---:|
| 前端 Vite | 优先 5173；Windows 保留时自动回退到 5241 |
| Spring Boot 后端 | 8080 |
| FastAPI 模型服务 | 8090 |
| MySQL | 3306 |
| Neo4j Browser | 7474 |
| Neo4j Bolt | 7687 |

## 本地脚本运行

`scripts/start-all.ps1` 默认使用 Spring Boot `demo` profile 和 H2 文件库，适合本机快速试用；Docker Compose 使用 MySQL + Neo4j，适合验收部署。

本地脚本约定：

- 模型服务固定使用 conda `MedRisk` 环境。脚本默认兼容 `D:\IDEA\Anaconda\envs\MedRisk\python.exe`，也会通过 `MEDRISK_CONDA_ROOT`、当前激活的 `MedRisk` 环境、常见 Anaconda/Miniconda 目录和 `conda env list` 发现其他电脑上的同名环境。
- 不创建、不激活 `medrisk_model_service\.venv`，也不从随机 `PATH` Python 启动。
- 模型服务启动前会在 conda `MedRisk` 环境内执行 `python -s -m pip install -r requirements.txt`。
- 后端通过 `mvn clean spring-boot:run -Dspring-boot.run.profiles=demo` 启动，避免旧构建产物中的重复 Flyway 迁移文件影响启动。
- 后端启动前会校验外部 Neo4j Bolt 地址，默认 `MEDRISK_NEO4J_URI=bolt://localhost:7687`。本地脚本只检查连接，不安装、不拉起、不停止 Neo4j；未就绪时会失败并提示先启动外部 Neo4j。
- 前端优先绑定 `127.0.0.1:5173`；如果该端口被 Windows/Docker/Hyper-V 保留，脚本会自动选择 `5241` 等可绑定端口，并打印实际 URL。Docker Compose 前端端口由 `.env` 的 `FRONTEND_PORT` 控制，示例默认 `5241`。
- 前端会检查 `vite`、`@vitejs/plugin-vue`、`jsdom`、`vue` 等依赖，不完整时自动执行 `npm install`。
- 后台进程的 stdin 被重定向到 `logs/empty.stdin`，脚本结束后当前 PowerShell 终端应可继续输入命令。
- 重复执行 `scripts/start-all.ps1` 时会按 5173、8080、8090 端口判断已有服务，避免重复拉起进程。

常用命令：

```powershell
.\scripts\start-all.ps1
.\scripts\stop-all.ps1
.\scripts\ci.ps1
```

`scripts/stop-all.ps1` 会清理记录在 `logs/medrisk-processes.json` 的进程、5173/8080/8090 端口进程，以及本项目目录下残留的 Vite/esbuild、Maven/Spring Boot、Uvicorn 相关进程。
它不会停止外部 Neo4j。若使用单独的 Docker Neo4j 容器，请按你的容器名称手动停止，例如 `docker stop medrisk-neo4j`。

可选的本地 Neo4j 快速启动命令：

```powershell
docker run --name medrisk-neo4j -p 7474:7474 -p 7687:7687 -e NEO4J_AUTH=neo4j/medrisk-neo4j -d neo4j:5.26-community
```

智能问答 LLM 通过 OpenAI-compatible 接口配置：`MEDRISK_LLM_BASE_URL`、`MEDRISK_LLM_API_KEY`、`MEDRISK_LLM_MODEL`、`MEDRISK_VISION_MODEL`。未配置 API Key 时，后端会基于图谱/疾病/病历检索结果生成演示兜底答案。

## 演示数据与图谱导入

后端启动时会幂等补齐基础演示包，不会重复插入已经存在的 `sourceRecordId`。管理员可在前端“数据源管理”中手动执行：

- “补齐演示数据”：写入关系库中的疾病、文档、合成病历和训练/评估数据集。
- “重建演示包”：补齐关系库数据并尝试向 Neo4j 写入结构化三元组。

演示包以公开来源元数据和本地合成内容组成，覆盖 MedlinePlus、Disease Ontology、HPO、UCI、NHANES、openFDA 链接型知识，以及 Synthea 风格合成病历。病历均标注“合成数据，非真实患者”，不包含真实姓名、证件号、电话、住址等身份信息。Neo4j 清理和重建只处理 `medriskSource=MedRisk` 的节点与关系，避免误删外部图谱数据。

文档、疾病、病历和图谱数据使用 `visibility` 做分级：`PUBLIC` 面向患者/医生/管理员，`DOCTOR_ONLY` 面向医生/管理员，`ADMIN_ONLY` 仅管理员，`DRAFT` 用于医生草稿和待发布内容。

换电脑部署本地开发环境时，需要先安装 Java 17、Maven、Node.js 20+ 和 Anaconda/Miniconda，并创建 conda 环境 `MedRisk`。如果 conda 根目录不在常见位置，可在当前 PowerShell 设置：

```powershell
$env:MEDRISK_CONDA_ROOT = "D:\Tools\miniconda3"
.\scripts\check-env.ps1
```

## 注意事项

- 系统只用于教学演示，不接入真实医院数据。
- 报告与模型输出必须保留免责声明。
- 示例数据应来自公开数据源元数据、许可证清晰的数据集或合成数据。
- 上传的数据集必须脱敏，且至少包含 `label` 目标列。
- 训练模型启用前应查看评估指标、混淆矩阵和模型反馈记录。
