# 开源发布清单

本清单用于准备 MedRisk AI 当前版本的公开源代码和公开文档。真实运行版本位于 `med_tencent:/home/med/MedRisk`，但该目录不是 git 仓库，并且可能包含运行态文件、个人报告材料或历史重复文件；开源发布包应以本地 git 工作区为准，在发布前只把必要的部署差异从远端核对回来。

## 发布范围

建议包含：

- `README.md`、`.env.example`、`docker-compose.yml`、`.dockerignore`、`docs/` 下公开说明文档。
- `medrisk_backend/` 的源码、迁移脚本、测试、`pom.xml`、Dockerfile 和资源配置模板。
- `medrisk_frontend/` 的源码、测试、品牌公开资源、`package.json`、`package-lock.json`、Dockerfile 和 Nginx/Vite 配置。
- `medrisk_model_service/` 的 FastAPI 源码、测试、Dockerfile、`requirements*.txt`。
- `scripts/` 下启动、停止、CI、环境检查、公开数据预处理脚本。
- `data/` 下小型教学演示 CSV 与说明文件。

不得包含：

- `.env`、任何真实数据库/Neo4j/JWT/SMTP/LLM/Hugging Face token。
- `AGENTS.md`、`.agents/`、AI 工具会话记录、自动化协作记录。
- `final_ret/`、`local_notes/`、`work_logs/`、`report_drafts/` 等个人工作日志、实习报告、截图和报告素材。
- `logs/`、`reports/`、`uploads/`、`tmp/`、`target/`、`dist/`、`node_modules/`。
- 训练产物、模型二进制、`models/training/`、`medrisk_backend/models/`、`*.joblib`、`*.pkl`、`*.pt` 等模型文件。
- 大体量或受控数据集，例如 `data/raw/`、`data/processed/*.csv`、MIMIC-IV 原始数据。
- 从远端目录直接复制出来的乱码文件名重复文档或 `.pytest_cache/`。

## 发布前核对

1. 先在远端只读确认运行目录状态：

   ```powershell
   ssh med_tencent "cd /home/med/MedRisk && docker compose ps && ls -la"
   ```

2. 如果远端有尚未同步的业务代码改动，只同步源码和公开配置；不要覆盖远端 `.env`，也不要把远端运行态目录打包进仓库。
3. 在本地确认发布文件列表：

   ```powershell
   git status --short
   git ls-files | rg "AGENTS|final_ret|local_notes|work_logs|ai_records|\\.env$|models/training|medrisk_backend/models"
   ```

   第二条命令应没有输出；如果有输出，先取消跟踪或移动到忽略目录。

4. 扫描敏感内容：

   ```powershell
   rg -n "AKIA|(^|[^A-Za-z0-9])sk-[A-Za-z0-9_-]{20,}|AIza[0-9A-Za-z_-]{20,}|BEGIN PRIVATE KEY|MEDRISK_LLM_API_KEY=\\S+|MEDRISK_MAIL_PASSWORD=\\S+|JWT_SECRET=\\S+" README.md docs scripts medrisk_backend medrisk_frontend medrisk_model_service .env.example
   ```

   允许出现环境变量名和占位符，不允许出现真实值。

5. 运行发布验证：

   ```powershell
   .\scripts\ci.ps1
   docker compose config --quiet
   ```

   若本机没有 Docker，可至少运行 `docker compose config` 或在远端执行 Compose 配置检查。

6. 检查许可证。当前文档说明了第三方引用边界，但正式公开前仍需要选择并加入项目 `LICENSE` 文件。

## 远端同步原则

- `med_tencent` 的运行目录是部署环境，不是发布源。远端可能存在 `.env`、上传文件、训练模型、日志、缓存、重复文档和报告草稿。
- 同步代码到远端时排除 `.env`、数据目录、上传目录、模型目录、日志目录、`target`、`dist`、`node_modules`、Docker 卷和个人材料。
- 远端 `.env` 只能按变量定向编辑，不要用本地示例文件整体覆盖。
- 只改前端时优先重建 `frontend`；改后端接口、审计、邮件、LLM、图谱时重建 `backend`；改模型训练、预测或依赖时重建 `model-service`。

## 工作日志和测试报告素材

公开仓库只保留可复现的工程事实：需求背景、架构、部署、接口、测试计划、第三方引用和开源边界。个人实习报告、工作日志、AI 工具使用记录、未公开截图和写作草稿应放在 gitignored 的 `local_notes/` 或 `final_ret/` 中。

可公开引用的报告线索来自：

- README 的项目定位、技术栈、核心功能和运行说明。
- `docs/部署说明.md` 的本地/远端部署、环境约束和故障复盘。
- `docs/接口说明.md` 的接口范围和权限边界。
- `docs/模型说明.md` 的模型训练、评估、数据来源和 LLM/GraphRAG 说明。
- `docs/测试计划.md` 的后端、前端、模型服务、集成和 CI 验证点。
- `docs/第三方引用说明.md` 的参考项目与许可证边界。
