# MedRisk AI 接口说明

统一响应结构：

```json
{"code":0,"message":"ok","data":{}}
```

## 本地访问地址

使用 `scripts/start-all.ps1` 本地启动后，默认访问地址如下：

| 服务 | 地址 |
|---|---|
| 前端 | 以 `start-all.ps1` 输出为准，通常 `http://localhost:5173`，端口被系统保留时为 `http://localhost:5241` |
| 后端 API | `http://localhost:8080/api` |
| 后端健康检查 | `http://localhost:8080/api/health` |
| 模型服务 | `http://localhost:8090` |
| 模型服务健康检查 | `http://localhost:8090/health` |
| Neo4j Browser | `http://localhost:7474` |
| Neo4j Bolt | `bolt://localhost:7687` |

本地启动脚本固定使用 conda `MedRisk` 环境运行模型服务，默认兼容 `D:\IDEA\Anaconda\envs\MedRisk\python.exe`，也支持在其他电脑上发现同名 conda 环境；不使用 `.venv` 或随机 `PATH` Python。后端以 Spring Boot `demo` profile 运行，并在启动前检查本机 Neo4j Bolt 地址，默认 `MEDRISK_NEO4J_URI=bolt://localhost:7687`。本地演示不会自动连接 `med_tencent`，远端 Neo4j 只供 Linux Docker Compose 内部服务使用，不推荐开放 `med_tencent:7687` 到公网。

接口排障时注意区分环境：本地 Windows 只使用 `bolt://localhost:7687`；`bolt://neo4j:7687` 是 Compose 服务名，只能在容器网络中解析。若图谱健康接口返回未连接，先确认本机 7687 是否有 Neo4j Bolt 监听。

Docker Compose 在远端只把 Neo4j Browser/Bolt 绑定到服务器本机 `127.0.0.1`，浏览器用户应通过前端端口访问系统，不直接访问 Neo4j。

`demo` profile 使用本地 H2 文件库，并关闭 Flyway checksum 校验以兼容历史演示库；Linux `med_tencent` 的 Docker Compose/MySQL 部署不使用该本地宽松校验。

`scripts/start-all.ps1` 只有在模型服务 `/health`、后端 `/api/health` 和前端页面均可访问后才输出启动完成；模型服务依赖安装输出写入 `logs/model-service-install.log`，不会把 pip 的 `Requirement already satisfied` 明细刷到终端。如果健康检查超时，应先查看脚本打印的日志尾部。

若本地前端登录返回 500，应先确认 `GET /api/health` 是否可达。该项目曾反复出现的启动故障是旧 `data/medrisk-demo` H2 库中的 Flyway checksum 与当前迁移文件不一致，导致后端启动失败；保留 `application-demo.yml` 的本地 H2 宽松校验并使用 `mvn clean spring-boot:run -Dspring-boot.run.profiles=demo` 可避免复发。

Linux 云服务器 `med_tencent` 使用 Docker Compose 部署时，前端容器 Nginx 会将浏览器侧 `/api/` 代理到后端容器 `backend:8080`。远端宿主机 Nginx 监听 80 并代理到前端映射端口 5241；后端 `8080` 仅作为健康检查和排障入口。

接口文档只记录公开路径、字段、权限和排障方法，不记录真实公网域名、远端 SSH 凭据、`.env` 值、邮箱授权码、LLM API Key、浏览器会话或 AI 工具使用记录。准备开源发布时请同时核对 [开源发布清单](开源发布清单.md)。

## 认证

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/auth/register/code` | 发送注册邮箱验证码，Request: `{ "email": "user@example.com" }` |
| POST | `/api/auth/register` | 注册用户 |
| POST | `/api/auth/login` | 登录并返回 JWT |
| POST | `/api/auth/password/forgot` | 发送忘记密码验证码，Request: `{ "email": "user@example.com" }` |
| POST | `/api/auth/password/reset` | 使用邮箱验证码重置密码，Request: `{ "email": "...", "emailCode": "123456", "newPassword": "..." }` |
| GET | `/api/auth/me` | 获取当前用户 |
| PUT | `/api/auth/me` | 更新当前用户姓名、邮箱、手机号，不能修改角色、状态、用户名 |
| POST | `/api/auth/me/avatar` | 上传当前用户头像，multipart 字段 `avatar` |
| POST | `/api/auth/me/password` | 校验旧密码后修改新密码 |
| POST | `/api/auth/logout` | 退出并写审计日志 |

登录成功后 JWT 会包含当前会话 `sid`。同一账号再次登录会刷新 `users.current_session_id`，旧 token 在下一次请求时返回 401，`message` 固定为 `该账号已在另一设备上登录，请重新登录`；退出登录只在当前 token 的 `sid` 匹配时清空会话，避免旧 token 清掉新会话。

|---|---|---|
| GET | `/api/history/predictions` | 当前用户预测历史；医生和管理员返回全量可见记录 |
| DELETE | `/api/history/predictions/{recordId}` | 删除预测历史；患者只能删除自己的记录，医生和管理员可删除全量可见记录，关联报告会同步删除 |

| GET | `/api/reports/{reportId}` | 报告详情 |
| GET | `/api/reports/{reportId}/download` | 下载 PDF |
| DELETE | `/api/reports/{reportId}` | 删除报告；只删除报告本身，不删除对应预测历史 |
| GET | `/api/admin/models` | 模型版本和指标 |
| POST | `/api/admin/models/{modelVersionId}/activate` | 启用指定模型版本 |
| DELETE | `/api/admin/models/{modelVersionId}` | 删除模型版本，并按顺序同步删除关联反馈、关联评估、对应训练任务和模型版本记录；返回删除数量 |
| GET | `/api/admin/audit-logs` | 审计日志，返回 `clientIp`；所有 HTTP 操作会自动记录请求 IP，后台任务无请求上下文时为空 |

`POST /api/auth/register` 现在必须携带 `emailCode`。验证码由 `/api/auth/register/code` 发送，验证通过后才创建账号；验证码默认 10 分钟有效、错误次数过多需重新获取。163 SMTP 应使用 `smtp.163.com:465 + SSL` 和第三方客户端授权码，接口错误会区分认证失败、SSL/端口错误、网络超时和未配置。

## 管理员训练管理

以下接口仅允许 `ADMIN` 角色访问。

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/admin/datasets` | 数据集列表 |
| POST | `/api/admin/datasets` | 上传 `.csv` 或 `.zip` 数据集，字段为 `name`、`diseaseType`、`description`、`file` |
| POST | `/api/admin/datasets/import-public` | 从 `MEDRISK_PUBLIC_DATASET_DIR` 的 `manifest.json` 幂等导入公开训练/评估 CSV，默认用于 CDC BRFSS、CDC NHANES，以及本地授权 MIMIC-IV 预处理输出；旧 manifest 中小于 1000 行的 UCI 项会被跳过 |
| PUT | `/api/admin/datasets/{id}` | 更新数据集名称、病种或说明 |
| DELETE | `/api/admin/datasets/{id}` | 删除数据集，并同步删除引用它的训练任务、评估记录和反馈；公开大数据原始 CSV 不会被物理删除 |
| POST | `/api/admin/datasets/prune-small` | 清理历史小样本演示/UCI 数据集，并同步删除相关训练任务、评估记录和反馈 |
| POST | `/api/admin/datasets/{id}/validate` | 重新校验数据集字段和样本数 |
| GET | `/api/admin/model-capabilities` | 查询模型服务支持的 `xgboost`、`lightgbm`、`catboost`、`tabpfn`、`tabicl` 能力和不可用原因 |
| GET | `/api/admin/training-jobs` | 训练任务列表 |
| POST | `/api/admin/training-jobs` | 创建训练任务，JSON 支持 `modelType`、`evaluationDatasetId` 和 `hyperparameters`，默认 `xgboost` |
| GET | `/api/admin/training-jobs/{id}/status` | 查询训练状态、进度、当前 loss 和指标 |
| GET | `/api/admin/training-jobs/{id}/history` | 查询训练曲线数据 |
| POST | `/api/admin/training-jobs/{id}/stop` | 请求终止训练任务 |
| DELETE | `/api/admin/training-jobs/{id}` | 删除训练任务记录和产物目录 |
| GET | `/api/admin/model-evaluations` | 模型评估记录 |
| POST | `/api/admin/model-evaluations` | 用指定数据集评估模型版本 |
| GET | `/api/admin/model-feedback` | 模型反馈列表 |
| POST | `/api/admin/model-feedback` | 新增模型反馈 |
| PUT | `/api/admin/model-feedback/{id}` | 更新模型反馈 |
| DELETE | `/api/admin/model-feedback/{id}` | 删除模型反馈 |

`POST /api/admin/training-jobs` 保留旧字段 `epochs`、`learningRate`、`testSize` 兼容旧前端，同时推荐使用模型专属 `hyperparameters`：

- `logistic_regression`：`cValue`、`maxIterations`、`classWeight`、`testSize`。
- `random_forest` / `extra_trees`：`nEstimators`、`maxDepth`、`classWeight`、`seed`、`testSize`。
- `hist_gradient_boosting`：`maxIterations`、`maxDepth`、`learningRate`、`regLambda`、`seed`、`testSize`。
- `xgboost` / `lightgbm` / `catboost`：`nEstimators`、`maxDepth`、`learningRate`、`subsample`、`colsampleBytree`、`regLambda`、`classWeight`、`testSize`；XGBoost 额外支持 `minChildWeight`。
- `tabpfn`：`maxTrainSamples`、`device`、`ensembleSize`、`version`、`testSize`，默认 `version=v2`。
- `tabicl`：`contextSize`、`maxTrainSamples`、`device`、`seed`、`testSize`。

`TrainingJobResponse` 和 `ModelVersionResponse` 会返回 `hyperparameters`、`evaluationDatasetName`、`evaluationDatasetSource`、`evaluationDatasetUrl`，用于前端展示训练实际使用参数和公开评估来源。

## 智能问答、文档与知识图谱

以下接口均需要 JWT；`/api/admin/**` 仅允许 `ADMIN`。患者不直接浏览文档、疾病、病历和图谱管理数据，只通过智能问答使用公开知识库检索结果；医生可浏览 `PUBLIC`、`DOCTOR_ONLY` 和自己提交的 `DRAFT`，管理员可访问全部。

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/documents` | 文档列表，支持 `keyword` |
| GET | `/api/documents/{id}` | 文档详情 |
| GET | `/api/documents/{id}/download` | 下载原始文档 |
| POST | `/api/admin/documents` | 上传 `txt/pdf/docx` 文档，multipart 字段 `title`、`file`、可选 `visibility` |
| POST | `/api/doctor/documents` | 医生提交文档草稿，强制保存为 `DRAFT` |
| PUT | `/api/admin/documents/{id}` | 更新文档标题、摘要、来源元数据或可见性 |
| DELETE | `/api/admin/documents/{id}` | 删除文档记录 |
| GET | `/api/admin/knowledge-graph/health` | Neo4j 连接健康检查，后端会即时验证 Driver 连通性 |
| POST | `/api/admin/knowledge-graph/rebuild` | 全量重建 MedRisk 写入的图谱数据，单篇失败不影响其他文档继续构建 |
| POST | `/api/admin/knowledge-graph/incremental` | 构建 `未构建` 与 `构建失败` 文档 |
| POST | `/api/admin/knowledge-graph/sync/{documentId}` | 同步单个文档，非空文档至少生成 50 个文档相关节点 |
| GET | `/api/admin/knowledge-graph/jobs` | 图谱构建任务列表 |
| GET | `/api/admin/knowledge-graph/search` | 图谱实体搜索，参数 `keyword` |
| GET | `/api/admin/knowledge-graph/visualization` | 管理员图谱可视化数据，支持 `keyword`、`nodeTypes`、`relationshipTypes`、`sourceName`、`visibility`、`limit` |
| GET | `/api/knowledge-graph/visualization` | 医生只读图谱可视化，按角色过滤可见数据 |
| GET | `/api/conversations` | 当前用户会话列表 |
| POST | `/api/conversations` | 创建会话，JSON 字段 `title` 可选 |
| GET | `/api/conversations/{id}` | 会话详情和历史消息 |
| DELETE | `/api/conversations/{id}` | 删除当前用户会话 |
| POST | `/api/conversations/{id}/messages` | 发送图片/多模态问题，multipart 字段 `question`，可选 `image`、`modelProfileId`、`reasoningEnabled`、`chatMode`、`outputImageRequested` |
| POST | `/api/conversations/{id}/messages/stream` | 文本流式问答，JSON 字段 `question`，可选 `modelProfileId`、`reasoningEnabled`、`chatMode=daily|medical`、`outputImageRequested` |
| GET | `/api/files/{bucket}/{objectKey}` | 受控访问上传文件、图片和文档 |

问答消息响应会额外返回 `usedModel`、`provider`、`fallbackUsed`、`chatMode`、`retrievalUsed`、`retrievalStatus`、`evidenceSources`、`generatedImages`，用于前端展示模型来源、是否使用本地兜底、检索状态、证据分区和生成图片。`chatMode=daily` 不执行知识检索；`chatMode=medical` 会检索图谱、疾病档案和病历案例。后端会在医学问答的检索和外部模型调用前执行平台范围判断：医疗健康、风险预测、医学知识库、病历/文档、知识图谱、模型训练评估和平台使用相关问题才会进入 LLM；明显无关问题直接返回范围提示，`usedModel=policy-guard`、`provider=medrisk-policy`、`fallbackUsed=true`，不会调用外部模型。

## 数据源与演示数据包

以下接口仅允许 `ADMIN` 角色访问。导入过程是幂等的，重复执行不会重复插入同一 `sourceRecordId` 的疾病、文档或病历；训练数据集由公开数据脚本、MIMIC-IV 本地预处理或管理员上传导入。

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/admin/data-seeds/status` | 查看演示包目标数量、当前数量、来源列表和最近导入记录 |
| POST | `/api/admin/data-seeds/import` | 补齐缺失的关系型演示数据，不强制写入 Neo4j |
| POST | `/api/admin/data-seeds/rebuild-demo-pack` | 补齐演示数据并尝试写入结构化图谱三元组 |

演示包字段统一包含 `sourceName`、`sourceUrl`、`sourceLicense`、`sourceRecordId`、`retrievedAt`、`visibility`。病历均为合成数据并带 `syntheticCase=true`，不包含真实患者身份字段。
